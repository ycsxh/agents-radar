## 3. Trend Signal Analysis

**Explosive attention is concentrated on "agent context infrastructure"**—tools that make codebases, documents, and institutional knowledge computationally accessible to AI agents. Three of today's top four trending projects ([codegraph](https://github.com/colbymchenry/codegraph), [Understand-Anything](https://github.com/Lum1104/Understand-Anything), [andrej-karpathy-skills](https://github.com/multica-ai/andrej-karpathy-skills)) address the same fundamental problem: LLM agents are context-window limited and tool-call expensive when operating on large, complex codebases. The market is coalescing around **pre-computed knowledge graphs** as the solution, with competitive differentiation in indexing granularity, local vs. cloud execution, and multi-agent compatibility.

A **new stack direction** is the formalization of "skills" as portable, declarative agent capabilities—evidenced by [dotnet/skills](https://github.com/dotnet/skills), [Anthropic-Cybersecurity-Skills](https://github.com/mukul975/Anthropic-Cybersecurity-Skills), and the Karpathy skills file. This represents a shift from imperative agent programming (code) to declarative capability specification (structured expertise artifacts), potentially enabling agent capability marketplaces and composable intelligence.

The timing correlates with **Claude Code's expanding ecosystem presence** and recent model releases (Kimi-K2.5, GLM-5, gpt-oss) that emphasize long-context and agentic capabilities. The community is building infrastructure *around* these models rather than *on top of* them—filling gaps in memory persistence, context management, and multi-agent coordination that foundation model providers have not addressed. The [MCP (Model Context Protocol)](https://github.com/ChromeDevTools/chrome-devtools-mcp) standardization across Chrome DevTools, vector databases, and memory systems suggests emergent protocol consolidation that could reduce fragmentation in agent tooling.
